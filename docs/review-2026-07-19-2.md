## 逐条评估

### #1 StreamRequest 属性污染 — ⏸ 暂缓（YAGNI）

- 现状核实：`StreamRequest` 仅有 `thinking_level: Option<&str>` 一个额外字段，远未到"污染"程度。Provider 在 `tau-ai` 层自行翻译为 vendor 参数。
- 评价：`provider_options: &Map<String, Value>` 会丧失类型安全，运行时错误难以排查。`trait ProviderOptionsTranslator` 在当前（仅 1 个扩展字段）阶段属过度设计。
- **决定**：暂缓。若 Phase 8 引入 ≥3 个 provider 专属字段时再统一为 `extras` map。当前在 `StreamRequest` 注释中记录此设计约束。

### #2 CodingSession 成长风险 — ⏸ 暂缓（记录设计原则）

- 现状核实：`CodingSession` 807 行 / 16 方法，compaction/naming/commands/repair 已拆为独立模块。
- 评价：引入 `trait SessionExtension` 或显式 `PrePromptHook` 管道在当前（无扩展）阶段属过早抽象。但方向正确——文档化原则即可。
- **决定**：在 `coding_session.rs` 注释中加设计约束："CodingSession 只做组合和委托，不写业务逻辑"。等 Phase 8 真长到 2000+ 行时再按模块拆分。

### #3 安全边界 — ✅ 采纳（PathPolicy trait）

- 现状核实：bash 工具无 sandbox（执行任意 LLM 生成的 shell 命令），read/write/edit 工具直接 join(cwd, path) 无路径校验。
- 实现：
  - `tau-agent/src/tool.rs` 新增 `PathPolicy` trait（`check(cwd, path) -> Result<PathBuf, String>`）
  - `NoPathPolicy`（默认）：允许所有路径，行为与原来完全一致
  - `RestrictedPathPolicy`：限制在项目根目录内（canonicalize + starts_with 检查）
  - `ReadExecutor`/`WriteExecutor`/`EditExecutor` 均持有 `Arc<dyn PathPolicy>`，execute() 中通过 `path_policy.check()` 解析路径
  - 构造函数 `new()` 默认使用 `NoPathPolicy`，`new_with_policy()` 可注入自定义策略
- 效果：默认行为不变（向后兼容），但 trait 边界已留出——未来加沙箱策略无需改动工具 executor 签名。

### #4 手写 Deserialize 缺乏 fuzz 测试 — ✅ 采纳（proptest）

- 现状核实：`AgentMessage` 和 `SessionEntry` 的 `Deserialize` 为手写实现（绕过 serde internally-tagged 不支持 `deny_unknown_fields`）。Golden 测试只能覆盖已知输入。
- 实现：
  - `tau-types` 新增 `dev-dependencies: proptest = "1"`
  - `message.rs` 新增 4 个 proptest：
    - `agent_message_roundtrips_user`（任意文本 → User 消息 → serialize → deserialize → 内容一致）
    - `agent_message_roundtrips_assistant`（任意文本 + 5 种 stop_reason 组合）
    - `agent_message_rejects_malformed_json`（任意字符串反序列化不 panic）
    - `agent_message_serialization_is_deterministic`
  - `session.rs` 新增 2 个 proptest：
    - `session_entry_never_panics`（任意 JSON 反序列化不 panic）
    - `message_entry_roundtrips_user_via_proptest`
- 效果：tau-types 测试从 4 → 11（+3 原有 + 4 新 proptest + 2 新 session proptest + 1 原有 session 测试），覆盖率从不覆盖非预期输入 → 覆盖任意输入安全。

### #5 Session 迁移链 — ⏸ 暂缓

- 现状核实：`migrate_session_entry(Value) -> Value` 处理 v1→当前一次迁移。当前仅 1 次迁移。
- 评价：v3 出现时重构为 `trait SchemaMigration` 链成本极低（10 行代码），现在做属过早优化。
- **决定**：暂缓。在 `jsonl.rs` 头部加 NOTE 注释提示未来重构方向。

### #6 可观测性 tracing — ⏸ 暂缓（已有部分基础）

- 现状核实：`tau-ai` 已用 `tracing::warn!` 记录 SSE 错误；`tau-cli` 已有 `tracing-subscriber` 初始化。但 `agent_loop`、`ToolExecutor::execute`、`retry` 未打 span。
- 评价：当前 `--verbose` + tracing 输出够用，且 `agent_loop` 的每次 tool execution 已有事件流（`ToolExecutionStart`/`ToolExecutionEnd`）可追踪。Span 是锦上添花而非缺失。
- **决定**：暂缓（低成本高回报但不阻塞）。
